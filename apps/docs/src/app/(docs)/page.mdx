# Introduction

`better-sslcommerz` is a TypeScript SDK for the [SSLCommerz](https://developer.sslcommerz.com/doc/v4/) gateway. It gives you a clean, strongly typed integration layer over SSLCommerz v4 so your payment flow is easier to build, test, and maintain.

Instead of manually building form-encoded payloads and handling loosely typed responses, you work with typed methods and consistent error handling.

## Why teams use it

- **End-to-end API coverage** - Create sessions, validate orders, process refunds, query transactions, and manage invoices.
- **Type-safe request and response models** - Method input/output is fully typed for safer integrations.
- **Schema-backed validation** - Request validation is built in, and response validation is available when you enable `validateResponse`.
- **IPN handling support** - Parse and validate incoming webhook payloads with `parseIpnPayload()`.
- **Operational controls** - Configurable timeout, custom `fetch`, and per-request abort support.
- **Structured errors** - Handle failures cleanly with `SslcommerzError`, `SslcommerzHttpError`, and `SslcommerzValidationError`.

## API coverage

The SDK maps directly to the key SSLCommerz v4 flows used in production:

- **Payment session** - `gwprocess/v4/api.php`
- **Order validation** - `validator/api/validationserverAPI.php`
- **Refund and transaction query** - `validator/api/merchantTransIDvalidationAPI.php`
- **Invoice create** - `gwprocess/v4/invoice.php`
- **Invoice status/cancel** - `validator/api/v4/`

This keeps your application code aligned with the official gateway behavior while avoiding repetitive low-level HTTP handling.

## Environments

| Environment | Base URL                           | Purpose                       |
| ----------- | ---------------------------------- | ----------------------------- |
| `sandbox`   | `https://sandbox.sslcommerz.com`   | Test payments, no real money  |
| `live`      | `https://securepay.sslcommerz.com` | Production, real transactions |

SSLCommerz requires **TLS 1.2 or higher**. Verify from your server with:

```bash
curl "https://sandbox.sslcommerz.com/public/tls/" -v
```
