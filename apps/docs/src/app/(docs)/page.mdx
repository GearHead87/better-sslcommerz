# Introduction

`better-sslcommerz` is a TypeScript-first client for the [SSLCommerz](https://developer.sslcommerz.com/doc/v4/) payment gateway — Bangladesh's most widely used payment processing platform.

It wraps the raw SSLCommerz REST API with full type safety, Zod-validated request/response schemas, and a clean two-namespace design so you can integrate payment flows without dealing with raw `fetch` calls or untyped responses.

## What you get

- **Type-safe client** — Every method input and output is fully typed. TypeScript catches mistakes before runtime.
- **Zod validation** — All request payloads are validated before sending. Enable `validateResponse` to also validate API responses against schemas.
- **Two API namespaces** — `client.core` for standard payments, IPN, validation, refunds, and transaction queries; `client.invoice` for invoice-based payment flows.
- **IPN payload parser** — `client.core.parseIpnPayload()` validates and types the raw POST body from SSLCommerz webhook callbacks.
- **Three error classes** — `SslcommerzError`, `SslcommerzHttpError`, and `SslcommerzValidationError` give you precise control over error handling.
- **Bring your own `fetch`** — Pass a custom `fetch` implementation (e.g. `undici`) via the `fetch` option.
- **Timeout support** — Built-in request timeout with `AbortSignal` propagation and per-request cancellation.
- **Sandbox & live environments** — Switch between environments with a single `environment` option.

## Package structure

The main package `better-sslcommerz` exposes two API namespaces on the client object:

```
client.core
  ├── createSession()          Initiate a payment session → redirect to gateway
  ├── validateOrder()          Validate a transaction after IPN / redirect
  ├── refundInitiate()         Initiate a refund for a completed transaction
  ├── refundStatus()           Check the status of an in-progress refund
  ├── transactionQueryBySession()   Look up a transaction by session key
  ├── transactionQueryByTranId()    Look up transactions by transaction ID
  └── parseIpnPayload()        Parse and validate an IPN webhook payload

client.invoice
  ├── createInvoice()          Create a shareable payment invoice link
  ├── invoiceStatus()          Check whether an invoice has been paid
  └── invoiceCancel()          Cancel a pending invoice
```

The `@better-sslcommerz/validators` package — which ships all Zod schemas and inferred TypeScript types — is re-exported directly from `better-sslcommerz`, so you only need one install.

## Environments

| Environment | Base URL                           | Purpose                       |
| ----------- | ---------------------------------- | ----------------------------- |
| `sandbox`   | `https://sandbox.sslcommerz.com`   | Test payments, no real money  |
| `live`      | `https://securepay.sslcommerz.com` | Production, real transactions |

SSLCommerz requires **TLS 1.2 or higher**. You can verify your server supports it with:

```bash
curl "https://sandbox.sslcommerz.com/public/tls/" -v
```
