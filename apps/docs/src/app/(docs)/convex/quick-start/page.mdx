# Quick Start

This guide takes you from a freshly installed component to a working payment flow. Complete [Installation](/convex/installation) first.

## 1. Create the component instance

Export a shared `SslCommerzConvex` instance from a helper module so you can import it across your Convex functions:

```ts
// convex/lib/sslcommerz.ts
import { SslCommerzConvex } from "@better-sslcommerz/convex";

import { components } from "../_generated/api";

export const sslcommerz = new SslCommerzConvex(components.sslcommerz);
```

Credentials are resolved automatically from the `SSLCOMMERZ_STORE_ID`, `SSLCOMMERZ_STORE_PASSWD`, and `SSLCOMMERZ_ENVIRONMENT` environment variables. To pass them explicitly instead:

```ts
export const sslcommerz = new SslCommerzConvex(components.sslcommerz, {
  storeId: process.env.SSLCOMMERZ_STORE_ID,
  storePasswd: process.env.SSLCOMMERZ_STORE_PASSWD,
  environment: "sandbox", // or "live"
});
```

## 2. Register HTTP routes

In your `convex/http.ts`, call `registerRoutes` to mount the four gateway callback routes:

```ts
// convex/http.ts
import { httpRouter } from "convex/server";

import { sslcommerz } from "./lib/sslcommerz";

const http = httpRouter();

sslcommerz.registerRoutes(http);

export default http;
```

This registers:

| Path                       | Trigger                                    |
| -------------------------- | ------------------------------------------ |
| `POST /sslcommerz/ipn`     | SSLCommerz IPN webhook                     |
| `POST /sslcommerz/success` | Customer redirect after successful payment |
| `POST /sslcommerz/fail`    | Customer redirect after payment failure    |
| `POST /sslcommerz/cancel`  | Customer redirect after cancellation       |

Copy the full IPN URL (`https://<your-deployment>.convex.site/sslcommerz/ipn`) into your SSLCommerz store settings as the IPN URL.

## 3. Initiate a payment session

Call `createPaymentSession` inside a Convex action. The method calls the SSLCommerz gateway and persists the session record in your database:

```ts
// convex/payments.ts
import { action, v } from "convex/server";

import { sslcommerz } from "./lib/sslcommerz";

export const initiatePayment = action({
  args: {
    tranId: v.string(),
    amount: v.number(),
    currency: v.string(),
    customerName: v.string(),
    customerEmail: v.string(),
    customerPhone: v.string(),
    customerAddress: v.string(),
    customerCity: v.string(),
    customerPostcode: v.string(),
  },
  handler: async (ctx, args) => {
    const deploymentUrl = "https://<your-deployment>.convex.site";
    const appUrl = "https://yourapp.com";

    const result = await sslcommerz.createPaymentSession(ctx, {
      tranId: args.tranId,
      totalAmount: args.amount,
      currency: args.currency,
      successUrl: `${appUrl}/payment/success`,
      failUrl: `${appUrl}/payment/fail`,
      cancelUrl: `${appUrl}/payment/cancel`,
      ipnUrl: `${deploymentUrl}/sslcommerz/ipn`,
      customerInfo: {
        name: args.customerName,
        email: args.customerEmail,
        phone: args.customerPhone,
        address1: args.customerAddress,
        city: args.customerCity,
        postcode: args.customerPostcode,
        country: "Bangladesh",
      },
      productInfo: {
        name: "Your Product",
        category: "general",
        profile: "general",
      },
    });

    return result.gatewayUrl;
  },
});
```

<Note>
  `tranId` must be unique for every payment attempt. SSLCommerz rejects
  duplicate transaction IDs. Generate one from your order ID, a timestamp, or a
  UUID.
</Note>

## 4. Redirect the customer

Call the action from your frontend and send the customer to the returned URL:

```ts
// In your frontend (e.g. a Next.js Server Action or Route Handler)
const gatewayUrl = await convex.action(api.payments.initiatePayment, {
  tranId: `ORDER_${orderId}_${Date.now()}`,
  amount: 500,
  currency: "BDT",
  customerName: "Rahim Uddin",
  customerEmail: "rahim@example.com",
  customerPhone: "01711234567",
  customerAddress: "House 10, Road 5, Dhanmondi",
  customerCity: "Dhaka",
  customerPostcode: "1205",
});

window.location.href = gatewayUrl;
```

The customer completes payment on the SSLCommerz page and is redirected to your `successUrl`, `failUrl`, or `cancelUrl`. The IPN webhook fires independently and the component handles verification automatically.

## 5. React to payment outcomes

Pass callbacks to `registerRoutes` to run business logic after each outcome. Each callback receives a Convex `ctx` (with `runQuery` and `runMutation`) and the stored session record:

```ts
sslcommerz.registerRoutes(http, {
  successRedirectUrl: "/dashboard/orders",
  failRedirectUrl: "/checkout?error=payment_failed",
  cancelRedirectUrl: "/checkout",

  onSuccess: async (ctx, session) => {
    if (!session) return;
    await ctx.runMutation(api.orders.markAsPaid, { tranId: session.tranId });
  },

  onFail: async (ctx, session) => {
    if (!session) return;
    await ctx.runMutation(api.orders.markAsFailed, { tranId: session.tranId });
  },

  onCancel: async (ctx, session) => {
    if (!session) return;
    await ctx.runMutation(api.orders.markAsCancelled, {
      tranId: session.tranId,
    });
  },

  onIpn: async (ctx, ipn, session) => {
    // ipn has already been verified against SSLCommerz by this point
    if (ipn.status === "VALID") {
      await ctx.runMutation(api.orders.confirmPayment, {
        tranId: ipn.tran_id,
        valId: ipn.val_id,
        bankTranId: ipn.bank_tran_id,
      });
    }
  },
});
```

See [HTTP Routes](/convex/http-routes) for the full lifecycle of each route and all `RegisterRoutesConfig` options.
