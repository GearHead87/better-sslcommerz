# Transaction Query

Two methods let you look up transaction details after a payment:

- `client.core.transactionQueryBySession()` — query by the `sessionkey` returned from `createSession`
- `client.core.transactionQueryByTranId()` — query by the `tran_id` you supplied to `createSession`

Both use the endpoint: `GET validator/api/merchantTransIDvalidationAPI.php`

These are useful for:

- Reconciling payments that were not confirmed via IPN
- Admin dashboards that display payment details
- Verifying payment state without relying on redirect data

---

## Query by session key

```ts
const result = await client.core.transactionQueryBySession(input, options?);
```

### Input

<Properties>
  <Property name="sessionkey" type="string" required>
    The `sessionkey` returned in the `createSession` response. Max 50
    characters.
  </Property>
</Properties>

### Example

```ts
import { sslcommerz } from "@/lib/sslcommerz";

const result = await sslcommerz.core.transactionQueryBySession({
  sessionkey: storedSession.sessionKey,
});

console.log(result.status); // e.g. "VALID"
console.log(result.tran_id); // your transaction ID
console.log(result.amount); // payment amount
```

## Query by transaction ID

```ts
const result = await client.core.transactionQueryByTranId(input, options?);
```

Querying by `tran_id` returns an **array** of transaction records because a customer may have attempted the same `tran_id` multiple times (e.g. multiple failed attempts before a success).

### Input

<Properties>
  <Property name="tran_id" type="string" required>
    The transaction ID you supplied to `createSession`. Max 50 characters.
  </Property>
</Properties>

### Example

```ts
import { sslcommerz } from "@/lib/sslcommerz";

const result = await sslcommerz.core.transactionQueryByTranId({
  tran_id: "ORDER_123456",
});

console.log(`Found ${result.no_of_trans_found} transaction(s)`);

// Find the successful transaction
const successful = result.element?.find(
  (tx) => tx.status === "VALID" || tx.status === "VALIDATED",
);

if (successful) {
  console.log("Paid:", successful.amount, successful.currency);
  console.log("Gateway:", successful.card_type);
}
```

### Response

<Properties>
  <Property name="APIConnect" type="string" optional>
    Internal SSLCommerz connection status.
  </Property>
  <Property name="no_of_trans_found" type="string" optional>
    The number of transaction records found for this `tran_id`.
  </Property>
  <Property name="element" type="array" optional>
    Array of transaction objects. Each element contains all the same fields as the `transactionQueryBySession` response, plus:

    - `bank_gw` — bank gateway name
    - `error` — failure reason if the transaction failed

  </Property>
</Properties>

### Element fields

Each item in `element` contains the same fields as the `transactionQueryBySession` response above, plus:

<Properties>
  <Property name="bank_gw" type="string" optional>
    Internal bank gateway identifier.
  </Property>
  <Property name="error" type="string" optional>
    Failure reason for failed transactions (max 255 characters).
  </Property>
</Properties>

## Status values

| Status        | Meaning                                     |
| ------------- | ------------------------------------------- |
| `VALID`       | Successful, confirmed transaction           |
| `VALIDATED`   | Valid, but already validated previously     |
| `FAILED`      | Payment failed at the gateway               |
| `CANCELLED`   | Customer cancelled the payment              |
| `EXPIRED`     | Payment session timed out                   |
| `UNATTEMPTED` | Customer did not select a payment method    |
| `PENDING`     | Payment awaiting completion (IB or delayed) |

<Note>
  For `PENDING` transactions, continue polling until the status changes.
  SSLCommerz will also send an IPN when the status resolves.
</Note>
